## 10. Christ Medical — Product & Design DNA

*(Mostly current and high-value; the soul of the product. Stack specifics reconciled in §9/§14.)*

**What it is:** offline-capable EMR for faith-based physician mission teams — fly in, set up in a church/community center, see 30+ patients/day, share the gospel. Not a hospital system, not a compliance product.
**Users:** attending physicians, triage nurses (Connie persona), mission directors, patients (printed multilingual discharge).

**Faith layer — NON-NEGOTIABLE UX:**
1. Pink **✝ Prayer note** on every SOAP encounter — always optional, never blocks signing, never removed.
2. Spiritual care is its **own full screen** (`/patients/[id]/spiritual`), not a modal.
3. **Cumulative spiritual record** aggregates across encounters (prayed-with ×3, gospel shared in Jan, pastor referral…).
4. Dashboard keeps **clinical and mission-impact stats in two separate panels** — never mixed.
5. **Team prayer log** — any care-team member can add a prayer note per encounter.
6. Discharge always has "closing prayer offered" + optional devotional/scripture given.
7. **Daily verse** set by the mission director (a `mission_config` row, not hardcoded).

**Logo concept:** the **Nehushtan** (bronze serpent on a pole, Numbers 21 → John 3:14) — reclaiming the Rod-of-Asclepius origin. Ancient, purposeful, not decorative.

**Design system:** Tailwind custom only (no MUI/Chakra/shadcn). DM Sans / DM Mono, two weights. No gradients, no heavy shadows, borders `0.5px`. Build every component in **Storybook** before pages.
- Primary purple `#3C3489` / mid `#534AB7` / light `#EEEDFE` / dim `#CECBF6`.
- Urgency: red `#A32D2D`/`#FCEBEB`, amber `#854F0B`/`#FAEEDA`, green `#3B6D11`/`#EAF3DE`.
- **SOAP color coding:** S=blue, O=green, A=amber, P=purple-light, ✝Prayer=pink (bg + left border each).
- Signature component **`PatientAvatar`** — age/sex silhouette + initials, urgency color, purple ✝ badge when spiritual care logged, grey silhouette+cross for unknown. Planned: StatusBadge, VitalsGrid, SOAPNote, AllergyBanner, MedicationList, IcdSearch, SpiritualCarePanel, PatientHeader (persistent in `/patients/[id]` layout route), QueueList, StatCard, LabResultRow.

**Realtime:** queue uses **SSE** from the API (no WebSockets, no Firebase in the live product — Firebase only on the offline wireframe-annotation tool). Keep PHI server-side; never log patient data to the browser console.

**Schema (key tables):** patients · encounters (SOAP fields + optional `prayer_note` + status draft/signed/discharged) · spiritual_care (prayer_offered, gospel_shared, faith_decision, bible_given, pastor_referral) · vitals · medications · allergies · lab_orders. Double Metaphone phonetic search via `fuzzystrmatch`.

**Wireframes:** `docs/wireframes/wireframes.html` (9 clickable screens) + annotated review version. Read the yellow UX notes before building a screen.
